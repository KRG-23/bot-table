# Munitorum – Politique de confidentialité / Privacy Policy

**Dernière mise à jour / Last updated:** 2026-02-04

---

## Français

### 1. Données collectées

Munitorum traite uniquement les données nécessaires au fonctionnement du bot :

- Identifiants Discord (ID serveur, ID canal, ID utilisateur)
- Pseudonyme/nom d’affichage Discord
- Contenu minimal des messages liés aux réservations (mentions de joueurs, jeu, date)
- Métadonnées techniques (horodatage, ID de message)

### 2. Finalités

Ces données sont utilisées pour :

- Créer et valider des réservations de tables
- Éviter les doublons (un joueur, une partie par soirée)
- Envoyer des notifications liées aux réservations
- Tenir un historique d’actions (audit interne du serveur)

### 3. Conservation

Les données sont conservées aussi longtemps que nécessaire au fonctionnement du bot et à la gestion des événements du club. Les sauvegardes de base de données sont conservées 1 mois.

### 4. Partage

Aucune donnée n’est vendue ni partagée à des tiers, en dehors des traitements nécessaires au fonctionnement de Discord et de l’hébergement du bot.

### 5. Sécurité

Les tokens et accès sensibles sont stockés en variables d’environnement. L’accès à la base est restreint aux administrateurs techniques du bot.

### 6. Vos droits

Pour toute demande d’accès, de correction ou de suppression des données, contactez les administrateurs du serveur Discord où Munitorum est utilisé.

### 7. Modifications

Cette politique peut être mise à jour. L’utilisation continue du bot vaut acceptation de la version en vigueur.

## English

### 1. Data collected

Munitorum processes only the data required to operate:

- Discord identifiers (server ID, channel ID, user ID)
- Discord username/display name
- Minimal message content related to reservations (player mentions, game, date)
- Technical metadata (timestamps, message IDs)

### 2. Purpose

This data is used to:

- Create and validate table reservations
- Prevent duplicates (one player, one match per evening)
- Send reservation-related notifications
- Keep an internal audit trail for the server

### 3. Retention

Data is kept as long as needed for bot operation and club event management. Database backups are retained for 1 month.

### 4. Sharing

No data is sold or shared with third parties beyond what is necessary to run Discord and the hosting environment.

### 5. Security

Tokens and sensitive access are stored in environment variables. Database access is restricted to bot administrators.

### 6. Your rights

For access, correction, or deletion requests, contact the administrators of the Discord server where Munitorum is used.

### 7. Changes

This policy may be updated. Continued use of the bot means acceptance of the current version.
